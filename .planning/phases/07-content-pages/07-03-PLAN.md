  <action>
    **src/app/api/admin/faqs/route.ts**
    POST only. Auth guard: createClient() + getUser(). Body: `{ category: string; question: string; answer: string; display_order?: number }`. Use createAdminClient() to insert into faqs. Return `{ faq }` on success (201), 401 on missing auth, 400 on validation fail (missing category/question/answer), 500 on DB error.

    **src/app/api/admin/faqs/[id]/route.ts**
    PUT and DELETE. Both require auth guard. PUT body: `{ category?: string; question?: string; answer?: string; display_order?: number }` — use createAdminClient().from('faqs').update(body).eq('id', id). DELETE — use createAdminClient().from('faqs').delete().eq('id', id). Both return `{ ok: true }` on success, 401 on missing auth, 404 if row not found.

    **src/app/(admin)/_components/FaqForm.tsx**
    `'use client'`. Two modes driven by optional `faq?: Faq` prop:
    - Create mode (faq undefined): empty fields, submits POST to /api/admin/faqs
    - Edit mode (faq defined): pre-filled fields, submits PUT to /api/admin/faqs/[faq.id]; shows Delete button with inline confirm (same pattern as ProductForm delete)

    Fields: category (text input with placeholder suggestions "Shipping / Products / Care / Orders"), question (text input), answer (textarea — plain text, no Tiptap), display_order (number input, optional, default 0).

    Toast pattern: `useState<{type: 'success'|'error'; message: string}|null>(null)` + setTimeout(3000). On successful create, call `router.push('/admin/faqs')` to return to list. On successful edit, show toast. On successful delete, call `router.push('/admin/faqs')`.

    **src/app/(admin)/admin/faqs/page.tsx**
    Server Component. Auth check + redirect. Fetch all FAQs: `createAdminClient().from('faqs').select('*').order('category').order('display_order')`. Render:
    - Page heading "FAQs"
    - `<FaqForm />` (create mode — no faq prop) in a card at the top labeled "Add New FAQ"
    - Below: table/list of existing FAQs grouped by category — each row shows category, question, and an "Edit" link to `/admin/faqs/[id]`

    **src/app/(admin)/admin/faqs/[id]/page.tsx**
    Server Component. Auth check + redirect. Next.js 15 async params: `const { id } = await params`. Fetch: `createAdminClient().from('faqs').select('*').eq('id', id).single()`. If error/null, call `notFound()`. Render `<FaqForm faq={faq} />` (edit mode).

    **src/components/faq/FaqAccordion.tsx**
    `'use client'`. Props: `{ faqs: Faq[] }`. State: `const [openId, setOpenId] = useState<string | null>(null)`.

    Group faqs by category:
    ```typescript
    const grouped = faqs.reduce<Record<string, Faq[]>>((acc, faq) => {
      acc[faq.category] ??= []
      acc[faq.category].push(faq)
      return acc
    }, {})
    ```

    Render each category as a `<section>` with an `<h2>` heading. Within each section, render the FAQs as an accordion:
    - Each item: `<button>` for the question (full-width, flex between question + chevron/+/−)
    - Click: `setOpenId(openId === faq.id ? null : faq.id)` — one-open-at-a-time
    - Answer: conditionally rendered `<p>` when `openId === faq.id`
    - Use `aria-expanded={openId === faq.id}` on the button for accessibility
    - Styling: `divide-y divide-stone-200` between items, `py-4` on button, gold `+`/`−` indicator

    For the open/close animation, use Tailwind v4's `grid` trick: wrap answer in `<div className={openId === faq.id ? 'grid-rows-[1fr]' : 'grid-rows-[0fr]'} ...>` with `overflow-hidden transition-all duration-200` — this animates height without JS measurement.
