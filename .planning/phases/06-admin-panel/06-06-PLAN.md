  <action>
    CREATE src/app/(admin)/admin/orders/page.tsx — Server Component:
    - Auth check (createClient + getUser + redirect)
    - Fetch all orders: createAdminClient().from('orders').select('id, created_at, paystack_reference, customer_name, total, status').order('created_at', { ascending: false })
    - Render: heading "Orders" + <OrdersTable orders={orders} />

    CREATE src/app/(admin)/_components/OrderStatusSelect.tsx — 'use client':
    - Props: orderId: string; currentStatus: string
    - Local state: status (initialised from currentStatus), saving: boolean
    - Renders a <select> with options: paid | processing | shipped | delivered
    - onChange: set saving=true, PATCH /api/admin/orders/{orderId} with { status: newValue }, on success update local status state, on error revert to previous status + show alert
    - Styled: select has bg-stone-800 border-stone-600; colour-coded options matching status badge colours
    - During save: show a small spinner next to the select; disable the select

    CREATE src/app/(admin)/_components/OrdersTable.tsx — 'use client':
    - Props: orders: Order[]
    - Local state: activeTab ('All' | 'Paid' | 'Processing' | 'Shipped' | 'Delivered')
    - Filter orders by status when activeTab !== 'All' (case-insensitive match)
    - Tab bar: "All ({total})" | "Paid ({n})" | "Processing ({n})" | "Shipped ({n})" | "Delivered ({n})"
    - Table columns: Order # | Customer | Date | Total (₦) | Status
    - Order #: show last 10 chars of paystack_reference, linked to /admin/orders/{id}
    - Customer: customer_name
    - Date: toLocaleDateString('en-NG', { day: 'numeric', month: 'short', year: 'numeric' })
    - Total: ₦{total.toLocaleString('en-NG')}
    - Status column: <OrderStatusSelect orderId={order.id} currentStatus={order.status} />
    - Empty state: "No orders with this status"
    - Responsive: on mobile, hide Date column (hidden md:table-cell)

    CREATE src/app/(admin)/admin/orders/[id]/page.tsx — Server Component (async):
    - Auth check
    - Fetch order + order_items: adminClient.from('orders').select('*, order_items(*)').eq('id', id).single()
    - Cast result via unknown to FullOrder type (same pattern as Phase 5 order confirmation)
    - If not found, return notFound()
    - Render two sections:

    Section 1 — Customer & Delivery:
    - Customer name, email, phone (WhatsApp), delivery address, delivery state
    - Status badge + <OrderStatusSelect> for inline update on the detail page too
    - Order date, total

    Section 2 — Line Items:
    - Table: Product | Variant | Thread Colour | Pack Qty | Unit Price | Line Total
    - Thread colour: show "—" if null (Tools products)
    - Footer row: subtotal + shipping cost + grand total
    - Same bg-stone-900 table styling as orders list
  </action>
