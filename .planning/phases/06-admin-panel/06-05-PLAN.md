  <name>Task 1: Install dnd-kit + create ImageUploader component</name>
  <files>
    src/app/(admin)/_components/ImageUploader.tsx
  </files>
  <action>
    INSTALL packages:
    npm install @dnd-kit/core@^6 @dnd-kit/sortable@^7 @dnd-kit/utilities@^3

    CREATE src/app/(admin)/_components/ImageUploader.tsx — 'use client':
    Props:
    - productId: string | undefined (undefined = new product, use a temp UUID)
    - initialImages: string[] (URLs of already-saved images, for edit mode)
    - onImagesChange: (urls: string[]) => void (called whenever order or set changes)

    Internal state:
    - images: Array<{ id: string; url: string }> — ordered list; id is a stable key for dnd-kit
    - uploading: boolean
    - error: string | null
    Initialise images from initialImages prop on mount.

    Upload logic (uploadFile function):
    - Import createClient from '@/lib/supabase/client' (browser client — NOT admin client)
    - Generate path: `${effectiveProductId}/${Date.now()}-${file.name}` where effectiveProductId = productId ?? tempId (useState(() => crypto.randomUUID()))
    - supabase.storage.from('product-images').upload(path, file, { upsert: false })
    - On success: supabase.storage.from('product-images').getPublicUrl(path) — append { id: crypto.randomUUID(), url: data.publicUrl } to images state
    - On error: set error state "Upload failed: {error.message}"
    - Limit: if images.length >= 5, show "Maximum 5 images reached" and skip upload

    Drop zone:
    - <div> with onDragOver (preventDefault), onDrop (extract files from event.dataTransfer.files, call uploadFile for each)
    - onClick: trigger hidden <input type="file" accept="image/*" multiple /> via ref
    - onChange on input: call uploadFile for each selected file
    - Visual: dashed border border-stone-600, bg-stone-800, "Drag images here or click to upload" text, image icon
    - During upload: show a spinner overlay on the drop zone

    dnd-kit sortable grid:
    - Wrap thumbnails in <DndContext sensors={sensors} collisionDetection={closestCenter} onDragEnd={handleDragEnd}>
    - sensors: useSensors(useSensor(PointerSensor))
    - <SortableContext items={images.map(i => i.id)} strategy={rectSortingStrategy}>
    - handleDragEnd: use arrayMove to reorder images state; call onImagesChange with new URL order
    - Each thumbnail: SortableThumb component using useSortable({ id: image.id })
      - Shows <img src={image.url}> (or <Image> if easier) 80x80 rounded, object-cover
      - X button (absolute top-right): removes image from state; call onImagesChange
      - cursor-grab, drag handle is the whole thumbnail
    - First thumbnail has a "Primary" label badge (gold, bottom-left) — this becomes products.image

    Call onImagesChange(images.map(i => i.url)) whenever images state changes (useEffect([images])).
  </action>
  <verify>npx tsc --noEmit</verify>
  <done>ImageUploader compiles with dnd-kit imports. Drop zone and SortableThumb are defined. No TypeScript errors.</done>
