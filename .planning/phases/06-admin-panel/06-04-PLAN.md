    CREATE src/app/api/admin/products/route.ts — POST handler:
    - Import createClient (server.ts) + createAdminClient (admin.ts)
    - Auth check: getUser() — return 401 if no user
    - Parse JSON body: { name, slug, description, material, is_featured, is_active, variants }
    - Validate: name non-empty, slug non-empty (auto-generate slug from name if not provided: name.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, ''))
    - Compute price_min/price_max from variants array: Math.min/max over all variant.price values
    - Set image: '/images/products/placeholder-bead.svg' (images are set by Plan 06-05)
    - Set images: []
    - adminClient.from('products').insert({...payload}) — return 201 with created product or 400/500 on error

    CREATE src/app/api/admin/products/[id]/route.ts — PUT + DELETE handlers:
    PUT handler:
    - Same auth check
    - Parse body: same fields as POST plus any image/images updates
    - Recompute price_min/price_max from updated variants
    - adminClient.from('products').update({...}).eq('id', id)
    - Return 200 with updated product

    DELETE handler:
    - Auth check
    - adminClient.from('products').delete().eq('id', id)
    - Return 200 { success: true }

    CRITICAL — variants JSONB shape: each variant must include price_tiers.
    Default for new variants created in admin: price_tiers: [{ qty: 1, price: variant.price }]
    This ensures the storefront's pack-size picker never crashes on variants.price_tiers.map().
    The API route should enforce this: if a variant in the incoming body lacks price_tiers, set it to [{ qty: 1, price: variant.price }].
