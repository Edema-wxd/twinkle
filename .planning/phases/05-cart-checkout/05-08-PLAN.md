Create `src/app/orders/[reference]/OrderPoller.tsx` with `'use client'` directive:

Props: `{ reference: string }`

State: `order: (Order & { order_items: OrderItem[] }) | null`, `timedOut: boolean`

useEffect on mount (deps: [reference]):
1. Create Supabase browser client
2. Set 30-second timeout: `setTimeout(() => setTimedOut(true), 30_000)`
3. Immediate fetch: `supabase.from('orders').select('*, order_items(*)').eq('paystack_reference', reference).single()` — if data found, clearTimeout + setOrder(data)
4. Realtime subscription: `.channel('order-' + reference).on('postgres_changes', { event: 'INSERT', schema: 'public', table: 'orders', filter: 'paystack_reference=eq.' + reference }, (payload) => { clearTimeout; setOrder(payload.new as Order) }).subscribe()`
5. Note: When order arrives via Realtime, it may not include order_items. Do a follow-up fetch for the full order with items when Realtime fires.
6. Cleanup: clearTimeout + supabase.removeChannel(channel)

Render:
- If order: render `<OrderConfirmationView order={order} />`
- If timedOut: render timeout message — "Your payment was received but we're still processing your order. Please contact us with your reference number: {reference}" with WhatsApp link. Show "contact us" link.
- If !order && !timedOut: render spinner/loading state — centered `font-body text-charcoal/60` "Processing your order..." with a simple CSS spinner (animate-spin border-gold border-t-transparent)

   ```typescript
   async function fetchOrderByReference(reference: string) {
     const supabase = createClient() // from @/lib/supabase/server
     // Use service role — orders has no RLS
     // Note: the server client in this project uses the anon key by default.
     // For service-role, construct a custom client:
     const { createClient: createServiceClient } = await import('@supabase/supabase-js')
     const serviceSupabase = createServiceClient(
       process.env.NEXT_PUBLIC_SUPABASE_URL!,
       process.env.SUPABASE_SERVICE_ROLE_SECRET!,
       { auth: { persistSession: false, autoRefreshToken: false, detectSessionInUrl: false } }
     )
     const result = await serviceSupabase
       .from('orders')
       .select('*, order_items(*)')
       .eq('paystack_reference', reference)
       .single()
     if (result.error || !result.data) return null
     return result.data
   }
   ```
4. If order found: render `<OrderConfirmationView order={order} />` directly (no client JS needed)
5. If order not found: render `<OrderPoller reference={reference} />` (client component handles polling)

No `notFound()` call — an unknown reference is a valid pending state, not a 404.

Export page metadata is not possible here since we don't know the order at static time — skip metadata for this page.
