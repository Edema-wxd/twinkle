<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Create Vercel project and configure environment variables</name>
  <instructions>
**This step requires you to manually create the Vercel project and set environment variables before the first build. NEXT_PUBLIC_* variables are baked at build time — setting them after the first build requires a redeploy.**

---

### Step 1: Create Vercel project

1. Go to https://vercel.com/new
2. Click "Import Git Repository"
3. Select the `twinkle` GitHub repository
4. Under "Configure Project":
   - Framework Preset: **Next.js** (auto-detected)
   - Root Directory: leave as `.` (default)
   - Build Command: leave as default (`next build`)
   - Output Directory: leave as default (`.next`)
5. **IMPORTANT: Do NOT click "Deploy" yet** — you must set environment variables first

---

### Step 2: Set environment variables (Production environment)

In the "Environment Variables" section of the configure screen (or in Project Settings > Environment Variables after project creation), add ALL of the following for the **Production** environment:

| Variable | Value |
|----------|-------|
| `NEXT_PUBLIC_SUPABASE_URL` | Your Supabase Project URL (from Supabase Dashboard > Project Settings > API) |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | Your Supabase publishable/anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Your Supabase service_role key (secret — never use NEXT_PUBLIC_ prefix) |
| `NEXT_PUBLIC_PAYSTACK_PUBLIC_KEY` | Paystack TEST public key: `pk_test_...` |
| `PAYSTACK_SECRET_KEY` | Paystack TEST secret key: `sk_test_...` (NO NEXT_PUBLIC_ prefix) |
| `NEXT_PUBLIC_SITE_URL` | `https://twinkle-locs.vercel.app` (use your actual Vercel URL once you know it) |
| `NEXT_PUBLIC_WHATSAPP_NUMBER` | Your real business WhatsApp number in E.164 format without + (e.g. `2349118888010`) |

**Do NOT add:**
- `NEXT_PUBLIC_PAYSTACK_SECRET_KEY` — this must never exist; the secret key must never reach the browser

---

### Step 3: Add Supabase redirect URL for staging domain

1. Go to: Supabase Dashboard > Authentication > URL Configuration
2. Under "Redirect URLs", click "Add URL"
3. Add: `https://twinkle-locs.vercel.app/**`
4. Save

This allows the admin session cookie to work on the staging domain.

---

### Step 4: Deploy

1. Back in Vercel, click "Deploy"
2. Watch the build logs — the build should complete in ~2-3 minutes
3. Once complete, note your staging URL (e.g. `https://twinkle-locs.vercel.app`)

**If the build fails:** Copy the error from the build logs and paste it here. Do not proceed to Task 3 until the build passes.

</task>
