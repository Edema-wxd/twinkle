<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Register Paystack test webhook URL</name>
  <instructions>
Before running the purchase flow smoke test, you must register the staging webhook URL with Paystack's **test mode** dashboard. Without this, payments will appear to succeed but no order will be created in Supabase.

**Steps:**

1. Go to: https://dashboard.paystack.com/#/settings/developer
2. Make sure you are in **Test Mode** (toggle in the top-right corner of Paystack dashboard)
3. Scroll down to "Webhook URL" under the Test API Keys section
4. Enter: `https://[your-staging-url]/api/webhooks/paystack`
   (Replace [your-staging-url] with your actual Vercel URL from 10-02, e.g. `twinkle-locs.vercel.app`)
5. Save

**Note:** Paystack has separate webhook URL fields for Test mode and Live mode. You are only configuring the test webhook here. The live webhook URL will be registered separately when you attach twinklelocs.com.
  </instructions>
  <resume-signal>Type "webhook registered" once you've saved the webhook URL in Paystack dashboard.</resume-signal>
</task>

- Admin login, order management, and product creation all worked (Task 4)

→ The site is ready for DNS cutover. Proceed to the DNS cutover checklist below.

**DNS cutover checklist (when ready):**

1. Upgrade Vercel plan to Pro before accepting real payments (Hobby plan restricts commercial use)
2. In Paystack Dashboard, switch to **Live Mode** and update API keys in Vercel:
   - `NEXT_PUBLIC_PAYSTACK_PUBLIC_KEY` → your `pk_live_...` key
   - `PAYSTACK_SECRET_KEY` → your `sk_live_...` key
   - Register live webhook URL: `https://twinklelocs.com/api/webhooks/paystack` in Paystack Live Mode
3. Update `NEXT_PUBLIC_SITE_URL` in Vercel to `https://twinklelocs.com`
4. In Vercel Dashboard > Project > Settings > Domains, add `twinklelocs.com` and `www.twinklelocs.com`
5. Update DNS records at your domain registrar to point to Vercel (Vercel provides the required A/CNAME records)
6. After DNS propagates (up to 48h, usually <1h), verify `https://twinklelocs.com` loads the site
7. Trigger a redeploy on Vercel so NEXT_PUBLIC_SITE_URL is baked with the production domain

**If ANY critical test failed:**
- Document the failures here
- Fix each issue (create gap closure tasks if needed)
- Re-run the failed smoke test before declaring ready
