Replace the generic integer quantity stepper in ProductDetailClient with a pack-size tier picker.
Customers pick a size (2mm / 4mm / 6mm) AND a pack size (e.g. 50 beads, 100 beads) — price updates
accordingly. selectedTierQty is exposed in state so Phase 5 can wire it directly to the cart.

Purpose: The quantity stepper (1–10) did not reflect how Twinkle Locs sells beads — in fixed pack
sizes at different price points. The tier picker matches the real product model from the CSV.

Output: Updated ProductDetailClient with working tier picker; price display driven by both selections.
</objective>

  <action>
    Rewrite ProductDetailClient to replace the integer quantity stepper with a pack-size tier picker.
    Key requirements:

    **State:**
    - Keep `selectedVariantId` — initialised to first in-stock variant (unchanged)
    - Replace `quantity` (integer) with `selectedTierQty` (number) — initialised to the first tier
      qty of the initially selected variant: `selectedVariant?.price_tiers[0]?.qty ?? 0`
    - When selectedVariantId changes, reset selectedTierQty to the new variant's first tier qty

    **Derived values:**
    - `selectedVariant` — unchanged (find by id)
    - `selectedTier` — find in `selectedVariant.price_tiers` where `tier.qty === selectedTierQty`
    - `displayPrice` — `selectedTier?.price ?? selectedVariant?.price ?? product.price_min`

    **Price display:**
    - Show `₦{displayPrice.toLocaleString()}` — updates when variant or tier changes
    - Below the price, show a small label: `"Pack of {selectedTierQty} beads"` (hidden for Shears)

    **Pack-size picker section** (replaces the integer stepper):
    - Section label: "Pack Size" (uppercase, same style as "Size")
    - Only render if the selected variant has more than 1 price tier
    - Render a button for each tier in `selectedVariant.price_tiers`:
      - Button text: `{tier.qty} beads`
      - Selected state: same border/bg treatment as size picker selected state
      - Unselected state: same border/text as size picker unselected state
    - Clicking a tier button sets `selectedTierQty` to that tier's qty

    **Remove:**
    - The old `quantity` state
    - The +/- stepper buttons and the `Math.max/min` clamp logic
    - The `setQuantity` function

    **Keep unchanged:**
    - Material badge, product name, description
    - Size picker (product.variants buttons)
    - Thread colour swatches section (still decorative placeholder, hidden for Tools)
    - Add to Cart button (still no-op with TODO comment for Phase 5)
    - `/* TODO Phase 5: wire to cart context */` — update comment to:
      `/* TODO Phase 5: wire selectedVariantId + selectedTierQty to cart context */`

    **Shears edge case:**
    - Shears has 1 tier: [{qty:1,price:3500}]
    - `selectedVariant.price_tiers.length <= 1` → do NOT render the pack-size picker section
    - Price label below price: hide for Shears (no "Pack of 1 beads" label)
    - The overall UI degrades gracefully with just the "Standard" size button and Add to Cart
