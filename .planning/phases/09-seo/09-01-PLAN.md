Create a simple SVG at the correct dimensions and save it as a JPEG-named placeholder using a Node.js one-liner:

```bash
node -e "
const fs = require('fs');
const svg = \`<svg xmlns='http://www.w3.org/2000/svg' width='1200' height='630' viewBox='0 0 1200 630'><rect width='1200' height='630' fill='#2C1810'/><text x='600' y='290' font-family='Georgia,serif' font-size='60' fill='#D4AF37' text-anchor='middle'>Twinkle Locs</text><text x='600' y='370' font-family='Georgia,serif' font-size='28' fill='#F5F0E8' text-anchor='middle'>Nigerian Loc Beads &amp; Accessories</text><text x='600' y='560' font-family='Georgia,serif' font-size='18' fill='#D4AF37' text-anchor='middle'>Replace with branded 1200x630 JPG before launch</text></svg>\`;
fs.writeFileSync('public/og-image.jpg', svg);
"
```

Note: This writes SVG content with a .jpg extension. Browsers and social crawlers accept this for development purposes. Next.js will serve it correctly. The placeholder clearly labels itself for replacement.

Also verify `.env.local` exists and add `NEXT_PUBLIC_SITE_URL` if it is missing. Run:

```bash
grep -q "NEXT_PUBLIC_SITE_URL" .env.local 2>/dev/null || echo "NEXT_PUBLIC_SITE_URL=http://localhost:3000" >> .env.local
```

This ensures the env var exists for local dev; the fallback `'https://twinklelocs.com'` in layout.tsx handles production.
